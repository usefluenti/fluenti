## Usage

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import fluenti from '@fluenti/vite-plugin'

export default defineConfig({
  plugins: [fluenti()],
})
```

## Options

```typescript
fluenti({
  framework: 'auto',     // 'vue' | 'react' | 'solid' | 'auto'
  catalogDir: 'src/locales/compiled',
  configPath: './fluenti.config.ts',
})
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `framework` | `'vue' \| 'react' \| 'solid' \| 'auto'` | `'auto'` | Framework detection mode |
| `catalogDir` | `string` | `'src/locales/compiled'` | Compiled catalogs directory |
| `configPath` | `string?` | — | Path to fluenti config |

## What it does

The plugin provides several sub-plugins, each implemented as a dedicated module:

1. **fluenti:virtual** — Resolves `virtual:fluenti/messages/{locale}` imports (`virtual-modules.ts`)
2. **fluenti:vt-transform** — Pre-transforms `v-t` directives in Vue SFCs (`vt-transform.ts`)
3. **fluenti:sfc-transform** — Transforms `<script setup>` in Vue SFCs (`sfc-transform.ts`)
4. **fluenti:tagged-template** — Transforms `` t`…` `` tagged templates (`tagged-template.ts`)
5. **fluenti:trans-transform** — Compile-time optimization for `<Trans>` components (`trans-transform.ts`)
6. **fluenti:solid-jsx** — SolidJS JSX-specific transforms (`solid-jsx-transform.ts`)
7. **fluenti:build-split** — Code-splitting transforms for production builds (`build-transform.ts`)
8. **fluenti:dev** — HMR support for catalog changes

Framework detection (`framework-detect.ts`) auto-detects the framework from project dependencies when `framework: 'auto'`.

## Script transforms

The plugin uses **AST scope analysis** to detect `t` bindings destructured from `useI18n()` and transforms tagged templates and function calls at build time. Only `t` variables that originate from `useI18n()` are transformed — other variables named `t` are left untouched.

**Vue mode:**

```
const { t } = useI18n()
t`Hello ${name}` → computed(() => __i18n.t('hash', { name: unref(name) }))
```

**React mode:**

```
const { t } = useI18n()
t`Hello ${name}` → __i18n.t('hash', { name })
```

React uses direct calls (no wrapper) since React's re-render cycle handles reactivity.

**Solid mode:**

```
const { t } = useI18n()
t`Hello ${name()}` → createMemo(() => __i18n.t('hash', { name: name() }))
```

### Legacy global `t` (deprecated)

Previous versions used a "magic global" `t` that required no import — the plugin injected it via regex matching. This approach is deprecated. To maintain backward compatibility, set `legacyGlobalT: true` in the plugin options:

```typescript
fluenti({ legacyGlobalT: true })
```

Migrate to `const { t } = useI18n()` — the scope-aware transform is more reliable and does not interfere with unrelated `t` variables.

## `<Trans>` compile-time optimization

The plugin also optimizes `<Trans>` components at build time. It statically extracts the message string from JSX children and pre-computes the message hash, injecting `__id`, `__message`, and `__components` props. This eliminates the runtime `extractMessage()` and `hashMessage()` cost — the optimization is fully transparent and requires no code changes.
