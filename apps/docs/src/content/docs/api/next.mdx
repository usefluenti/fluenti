## `withFluenti(options?)`

Creates a Next.js config wrapper that adds the Fluenti webpack/turbopack loader and dev-mode auto-compilation.

```typescript
// next.config.ts
import { withFluenti } from '@fluenti/next'

export default withFluenti({
  locales: ['en', 'ja', 'ar'],
  catalogDir: './locales',
  compileOutDir: './src/locales/compiled',
  format: 'po',
  serverModule: './src/lib/i18n.server',
})({
  reactStrictMode: true,
})
```

### `FluentiNextOptions`

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `catalogDir` | `string?` | `'./locales'` | Directory containing source translation catalogs (PO/JSON) |
| `compileOutDir` | `string?` | `'./src/locales/compiled'` | Output directory for compiled JS catalogs |
| `sourceLocale` | `string?` | `'en'` | Source locale code |
| `locales` | `string[]?` | `[sourceLocale]` | List of locale codes to compile |
| `format` | `'json' \| 'po'?` | `'po'` | Catalog file format |
| `autoCompile` | `boolean?` | `true` | Auto-compile catalogs and watch for changes in dev mode |
| `serverModule` | `string?` | â€” | Path to the server i18n module. Enables `t``\`\`` in Server Components |

### Return value

```typescript
(nextConfig?: NextConfig) => NextConfig
```

A function that wraps your Next.js config. Adds:
- A webpack rule with the Fluenti loader for `.tsx`, `.jsx`, `.ts`, `.js` files
- Turbopack rules via `turbopack.rules` (Next.js 15.3+)
- File watching in dev mode when `autoCompile` is enabled

### Chaining

The wrapper preserves existing config, including custom `webpack` functions:

```typescript
export default withFluenti({ locales: ['en', 'ja'] })({
  webpack(config, options) {
    // Runs after the Fluenti loader is added
    return config
  },
})
```

## `@fluenti/next/server`

Server-side i18n module for Next.js App Router.

### `configureServerI18n(config)`

Wraps `createServerI18n` from `@fluenti/react/server` with two enhancements:

1. **`setLocale()` is async** â€” pre-loads messages into the per-request `React.cache`, so downstream `t``\`\`` calls resolve synchronously
2. **Registers a singleton** â€” the loader's `__getServerI18n()` reads from this singleton

```typescript
// src/lib/i18n.server.ts
import { configureServerI18n } from '@fluenti/next/server'

export const { setLocale, getI18n, Trans, Plural, DateTime, NumberFormat } =
  configureServerI18n({
    loadMessages: (locale) => import(`@/locales/compiled/${locale}`),
    fallbackLocale: 'en',
    resolveLocale: async () => {
      const { cookies } = await import('next/headers')
      return (await cookies()).get('locale')?.value ?? 'en'
    },
  })
```

### Return value (`NextServerI18n`)

Same as `ServerI18n` from `@fluenti/react/server`, except:

| Property | Type | Difference |
|----------|------|------------|
| `setLocale` | `(locale: string) => Promise<void>` | **Async** â€” pre-loads messages. Must be `await`ed in the layout. |

All other properties (`getI18n`, `Trans`, `Plural`, `DateTime`, `NumberFormat`, `getI18nSync`) are unchanged.

## Webpack Loader

The Fluenti loader transforms React source files at build time. It detects whether a file is a Server Component or Client Component and applies the appropriate transform.

### Server Components (no `'use client'`)

When `serverModule` is configured, files without `'use client'` use the server transform:

```tsx
// Input
const msg = t`Hello ${name}`

// Output
import { __getServerI18n } from '@fluenti/next/server'
const msg = __getServerI18n().t('Hello {name}', { name: name })
```

`__getServerI18n()` is a synchronous function â€” not a hook. It reads the pre-loaded i18n instance from `React.cache` (per-request). No `await`, no `useContext`.

#### Auto-injected components

`Trans`, `Plural`, `DateTime`, and `NumberFormat` components in server files are automatically rewritten to singleton versions from `@fluenti/next/server`:

```tsx
// Input â€” no imports needed
export default function Page() {
  return <Trans>Read the <a href="/docs">documentation</a>.</Trans>
}

// Output
import { __Trans } from '@fluenti/next/server'
export default function Page() {
  return <__Trans>Read the <a href="/docs">documentation</a>.</__Trans>
}
```

If a component is already imported (e.g. from `@/lib/i18n.server`), the loader leaves it untouched.

### Client Components (`'use client'`)

Files with `'use client'` use the client transform:

```tsx
// Input
'use client'
const msg = t`Hello ${name}`

// Output
'use client'
import { __useI18n } from '@fluenti/react'
const __i18n = __useI18n()
const msg = __i18n.t('Hello {name}', { name: name })
```

### Tagged template: `t``\`\``

```tsx
// Input
const msg = t`Hello ${name}`
const info = t`You have ${count} items in ${category}`

// Server output
const msg = __getServerI18n().t('Hello {name}', { name: name })
const info = __getServerI18n().t('You have {count} items in {category}', { count: count, category: category })
```

#### Expression handling

| Expression | ICU placeholder | Value |
|-----------|----------------|-------|
| `${name}` (identifier) | `{name}` | `name` |
| `${user.name}` (property access) | `{name}` | `user.name` |
| `${a + b}` (complex expression) | `{0}` (positional) | `a + b` |

### Function call: `t()`

```tsx
// Input
const msg = t('Welcome to {app}', { app: 'MyApp' })

// Server output
const msg = __getServerI18n().t('Welcome to {app}', { app: 'MyApp' })
```

Only standalone `t()` calls are transformed. Method calls like `i18n.t()` or `$t()` are left untouched.

### File filtering

The loader only processes files that:
- Match `.tsx`, `.jsx`, `.ts`, or `.js` extensions
- Are **not** inside `node_modules`
- Contain `` t`\`` ``, `t()`, or component tags (`Trans`, `Plural`, `DateTime`, `NumberFormat`) (quick regex check â€” files without these patterns are passed through unchanged)

## Auto-Compilation

When `autoCompile` is `true` (default) and `next dev` is running:

1. **Initial compile** â€” catalogs are compiled on dev server startup
2. **File watching** â€” the `catalogDir` is watched for changes to `.po` or `.json` files
3. **Re-compile** â€” on file change, catalogs are re-compiled to `compileOutDir`
4. **HMR** â€” Next.js detects the output file change and triggers a hot module reload

This eliminates the need to manually run `fluenti compile` during development.

### Production builds

Auto-compilation only runs in `next dev`. For production, add `fluenti compile` to your build script:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "fluenti compile && next build"
  }
}
```

## Turbopack Support

The plugin registers the loader for turbopack using the `turbopack.rules` config format (Next.js 15.3+):

```typescript
// Automatically added by withFluenti():
{
  turbopack: {
    rules: {
      '*.tsx': { loaders: ['@fluenti/next/loader'], as: '*.js' },
      '*.jsx': { loaders: ['@fluenti/next/loader'], as: '*.js' },
      '*.ts':  { loaders: ['@fluenti/next/loader'], as: '*.js' },
      '*.js':  { loaders: ['@fluenti/next/loader'], as: '*.js' },
    },
  },
}
```

If you already have a `turbopack` config, the plugin merges its rules (your rules take precedence).
