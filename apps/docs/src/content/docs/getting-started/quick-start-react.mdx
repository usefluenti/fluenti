import { Steps, Tabs, TabItem, Aside } from '@astrojs/starlight/components'

<Steps>

1. **Install packages**

   ```bash
   pnpm add @fluenti/core @fluenti/react
   pnpm add -D @fluenti/cli @fluenti/vite-plugin
   ```

2. **Configure Vite**

   ```typescript
   // vite.config.ts
   import { defineConfig } from 'vite'
   import react from '@vitejs/plugin-react'
   import fluenti from '@fluenti/vite-plugin'

   export default defineConfig({
     plugins: [
       fluenti({ framework: 'react' }),
       react(),
     ],
   })
   ```

   The plugin enables compile-time transforms — `t` tagged templates and `t()` calls are rewritten at build time, so there's no runtime parsing overhead.

3. **Create `fluenti.config.ts`**

   ```typescript
   // fluenti.config.ts
   export default {
     sourceLocale: 'en',
     locales: ['en', 'zh-CN'],
     catalogDir: './locales',
     format: 'po',
     include: ['./src/**/*.{tsx,ts,jsx}'],
     compileOutDir: './src/locales/compiled',
   }
   ```

4. **Write your component — use tagged templates or `t()` with source text**

   <Tabs>
     <TabItem label="Tagged template (recommended)">
       ```tsx
       // src/Home.tsx
       export function Home() {
         const name = 'World'

         return (
           <div>
             <h1>{t`Welcome to my app`}</h1>
             <p>{t`Hello, ${name}!`}</p>
           </div>
         )
       }
       ```

       No imports needed — the Vite plugin auto-injects `useI18n()` and transforms the tagged template at build time:

       ```
       t`Hello, ${name}!` → __i18n.t('Hello, {name}!', { name })
       ```
     </TabItem>
     <TabItem label="useI18n hook">
       ```tsx
       // src/Home.tsx
       import { useI18n } from '@fluenti/react'

       export function Home() {
         const { i18n, setLocale } = useI18n()

         return (
           <div>
             <h1>{i18n.t('Welcome to my app')}</h1>
             <p>{i18n.t('Hello, {name}!', { name: 'World' })}</p>

             <button onClick={() => setLocale('en')}>English</button>
             <button onClick={() => setLocale('zh-CN')}>中文</button>
           </div>
         )
       }
       ```
     </TabItem>
   </Tabs>

   Both approaches write **source text directly in code** — no manual key management.

5. **Extract messages**

   ```bash
   npx fluenti extract
   ```

   This scans your source files and generates catalog files in `./locales/`:
   - `locales/en.po` — source messages (auto-filled)
   - `locales/zh-CN.po` — empty translations to fill in

6. **Add translations**

   Open `locales/zh-CN.po` and fill in `msgstr` fields:

   ```po
   msgid "Welcome to my app"
   msgstr "欢迎使用我的应用"

   msgid "Hello, {name}!"
   msgstr "你好，{name}！"
   ```

7. **Compile messages**

   ```bash
   npx fluenti compile
   ```

   This generates optimized JS modules in `src/locales/compiled/` — each message becomes a `/* @__PURE__ */` annotated export that bundlers can tree-shake. Unused translations are eliminated from production builds.

8. **Initialize from compiled output**

   ```tsx
   // src/main.tsx
   import { createRoot } from 'react-dom/client'
   import { I18nProvider } from '@fluenti/react'
   import { Home } from './Home'
   import en from './locales/compiled/en'
   import zhCN from './locales/compiled/zh-CN'

   function App() {
     return (
       <I18nProvider
         locale="en"
         fallbackLocale="en"
         messages={{ en, 'zh-CN': zhCN }}
       >
         <Home />
       </I18nProvider>
     )
   }

   createRoot(document.getElementById('root')!).render(<App />)
   ```

</Steps>

## What the compiler does

Fluenti's Vite plugin handles two compile-time transforms for React:

| Source code | Compiled output | What happens |
|---|---|---|
| `` t`Hello, ${name}!` `` | `__i18n.t('Hello, {name}!', { name })` | Tagged template → hash lookup, `useI18n()` auto-injected |
| `i18n.t('Hello, {name}!', { name })` | `i18n.t('a1b2c3', { name })` | Source text → hash ID, zero parsing at runtime |

At runtime, `i18n.t()` is an **O(1) hash lookup** into the compiled catalog — no message parsing, no ICU compilation, no `useMemo` needed.

## Rich text with `<Trans>`

```tsx
import { Trans } from '@fluenti/react'

<Trans>Click <a href="/next">here</a> to continue.</Trans>
<Trans>This is <strong>important</strong> information.</Trans>
```

Write JSX naturally. The markup structure is preserved in the catalog so translators see positional tags like `<0>here</0>`, not raw HTML.

## Plurals with `<Plural>`

```tsx
import { Plural } from '@fluenti/react'

<Plural value={count} zero="No items" one="# item" other="# items" />
```

## Next.js and React Server Components

For **Next.js** projects, use [`@fluenti/next`](/getting-started/quick-start-nextjs/) instead of the Vite plugin. It unlocks:

- **`t``\`\`` in Server Components** — same tagged template syntax works everywhere, no `await getI18n()` needed
- **Zero client JS for static content** — translations resolve on the server, nothing shipped to the browser
- **Auto-compilation in dev** — catalogs recompile on file change, no manual `fluenti compile` during development
- **Turbopack support** — loader works with both webpack and turbopack

```tsx
// src/app/page.tsx — Server Component, zero imports, zero await
export default function Home() {
  const name = 'World'

  return (
    <div>
      <h1>{t`Welcome to my app`}</h1>
      <p>{t`Hello, ${name}!`}</p>
    </div>
  )
}
```

The loader detects Server Components (no `'use client'`) and transforms `t``\`\`` to `__getServerI18n().t(...)` — a synchronous call into the pre-loaded per-request cache. No async, no hook, no manual import.

See the [Next.js Quick Start](/getting-started/quick-start-nextjs/) for a full setup guide.

## Framework compatibility

`@fluenti/react` works with any React-based framework:

- **React SPA** (Vite) — full Vite plugin support with compile-time transforms
- **React Router** — client-side routing with locale persistence
- **Remix** — file-based routing with Vite SPA mode
- **Next.js** — App Router with RSC support via [`@fluenti/next`](/getting-started/quick-start-nextjs/)
