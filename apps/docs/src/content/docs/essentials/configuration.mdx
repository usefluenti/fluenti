title: Configuration
description: Configuring fluenti.config.ts for your project
---

Fluenti uses a `fluenti.config.ts` file at your project root. It is auto-loaded via jiti — no build step needed.

## Example

```typescript
// fluenti.config.ts
import { defineConfig } from '@fluenti/cli'

export default defineConfig({
  sourceLocale: 'en',
  locales: ['en', 'zh-CN', 'ja'],
  catalogDir: './locales',
  format: 'po',
  include: ['./src/**/*.vue', './src/**/*.tsx', './src/**/*.jsx', './src/**/*.ts'],
  exclude: ['./src/**/*.test.*'],
  compileOutDir: './src/locales/compiled',
  fallbackChain: { 'zh-TW': ['zh-CN', 'en'], '*': ['en'] },
  splitting: 'dynamic',
  defaultBuildLocale: 'en',
})
```

`defineConfig` is an identity function that provides full type inference and IDE autocompletion for all config options.

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `sourceLocale` | `string` | `'en'` | The locale used in source code |
| `locales` | `LocaleDefinition[]` | `['en']` | All target locales. Each entry can be a string (`'en'`) or an object (`{ code: 'en', name: 'English' }`) |
| `catalogDir` | `string` | `'./locales'` | Directory for translation catalogs |
| `format` | `'json' \| 'po'` | `'po'` | Catalog file format. PO is recommended — compatible with Poedit, Weblate, Crowdin and other translation tools |
| `include` | `string[]` | `['./src/**/*.vue', './src/**/*.tsx', './src/**/*.jsx']` | Glob patterns for source files |
| `exclude` | `string[]` | `['./src/**/*.test.*']` | Glob patterns to exclude |
| `compileOutDir` | `string` | `'./src/locales/compiled'` | Output directory for compiled messages |
| `fallbackChain` | `Record<string, string[]>` | `{}` | Locale fallback chains |
| `splitting` | `'dynamic' \| 'static' \| false` | `false` | Code-splitting strategy |
| `defaultBuildLocale` | `string?` | — | Default locale embedded in the bundle |
| `devWarnings` | `boolean?` | — | Enable development-mode warnings for missing translations and format errors |
| `strictBuild` | `boolean?` | — | Fail the build if any translations are missing |
| `namespaceMapping` | `Record<string, string>?` | — | Map namespace prefixes to catalog subdirectories |
| `externalCatalogs` | `Array<{ package: string; catalogDir: string }>?` | — | Load catalogs from external npm packages |
| `strictThreshold` | `number?` | — | Minimum translation percentage required (used by `fluenti stats`) |
| `extends` | `string?` | — | Path to parent config file for [monorepo inheritance](/advanced/scaling/#monorepo-configuration) |
| `defaultLocale` | `string?` | — | Default locale for routing/detection (defaults to `sourceLocale`) |
| `parallelCompile` | `boolean?` | `false` | Enable [parallel compilation](/advanced/scaling/#parallel-compilation) using worker threads |
| `idGenerator` | `(message: string, context?: string) => string` | — | Custom message ID hash function |
| `catalogExtension` | `string?` | `'.js'` | File extension for compiled catalog output files |
| `devAutoCompile` | `boolean?` | `true` | Auto extract+compile in dev mode |
| `buildAutoCompile` | `boolean?` | `true` | Auto extract+compile before production build |
| `devAutoCompileDelay` | `number?` | `500` | Debounce delay in ms for dev auto-compile |
| `onBeforeCompile` | `() => boolean \| void \| Promise<...>` | — | [Lifecycle hook](/advanced/scaling/#build-lifecycle-hooks) called before auto-compile |
| `onAfterCompile` | `() => void \| Promise<void>` | — | [Lifecycle hook](/advanced/scaling/#build-lifecycle-hooks) called after auto-compile |

## PO vs JSON

| | PO | JSON |
|---|---|---|
| **Best for** | Teams with translators, Crowdin/Weblate | Solo devs, programmatic manipulation |
| **Tooling** | Poedit, Crowdin, Weblate, Lokalise | Any text editor, `jq` |
| **Comments** | First-class `#. translator comment` | Not supported |
| **Recommendation** | Default choice for production | Quick prototyping |

## Compiled output

Compiled catalogs are build artifacts. Regenerate them in CI instead of committing:

```txt
# .gitignore
locales/compiled/
```

For the full API reference, see [@fluenti/cli](/api/cli/) and [@fluenti/vite-plugin](/api/vite-plugin/).

## Monorepo Configuration

Use the `extends` option to inherit from a shared base config:

```typescript
// packages/app-web/fluenti.config.ts
export default defineConfig({
  extends: '../../fluenti.config.ts',
  include: ['./src/**/*.tsx', './src/**/*.ts'],
  splitting: 'dynamic',
})
```

Child configs inherit all options from the parent and can override any field. Relative paths in the parent are automatically rebased to the child's directory.

See [Scaling & Enterprise](/advanced/scaling/#monorepo-configuration) for a full monorepo setup guide.

## Performance Options

Enable parallel compilation and incremental caching for large projects:

```typescript
export default defineConfig({
  // ...
  parallelCompile: true,         // Worker threads for 5+ locales
  devAutoCompileDelay: 1000,     // Increase debounce for large codebases
})
```

Use `--no-cache` flags on `extract` and `compile` to force full rebuilds when needed. See [Scaling & Enterprise](/advanced/scaling/#incremental-caching) for details.

## Lifecycle Hooks

Hook into the auto-compile process for custom build integration:

```typescript
export default defineConfig({
  // ...
  onBeforeCompile: () => {
    // Return false to skip compilation
    console.log('Compiling translations...')
  },
  onAfterCompile: async () => {
    // Post-compile: notifications, uploads, validation
  },
})
```

See [Scaling & Enterprise](/advanced/scaling/#build-lifecycle-hooks) for details.
